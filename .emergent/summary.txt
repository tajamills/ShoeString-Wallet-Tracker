**Key Technical Concepts**
- **Per-Asset FIFO Calculation**: The core concept to fix the main bug. It involves grouping all transactions by their asset symbol before applying the FIFO algorithm to ensure buys and sells of the same asset are matched correctly.
- **Exchange Deposit Address Heuristics**: A pattern-matching approach to identify transient wallets. It checks for a combination of near-zero balance, total-in being almost equal to total-out, and transactions showing a rapid sweep pattern (receive followed quickly by a send of a similar amount).
- **Performance Optimization via Batching & Pagination**:
    - **Bulk API Calls**: Using API endpoints that return large sets of data in a single request (e.g., CryptoCompare's ) instead of making hundreds of individual requests in a loop.
    - **Pagination**: Handling large datasets from RPC endpoints (like Alchemy) by fetching them in sequential pages until all data is retrieved.
- **API Response Throttling**: A strategy to keep the frontend responsive when dealing with large backend datasets. The API computes on the full dataset (e.g., 12,000 transactions) but only returns a small, manageable subset (e.g., 100) for UI display, along with a total count.
